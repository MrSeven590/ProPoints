# 优化考核功能 UI 和交互

## Goal

优化考核功能的用户体验,使保存按钮状态更合理,分数信息展示更清晰直观。

## Requirements

### 1. 保存按钮状态优化
- 默认状态: 灰色不可点击
- 可点击条件: 当考核分数输入框有变更时才可点击
- 保存后: 恢复为灰色不可点击状态

### 2. 保存后分数显示
在对应人员下方显示三项分数信息:
- **原始分数**: 该人员的原始工分
- **考核分数**: 本次考核扣除的分数
- **最终分数**: 原始分数 - 考核分数

### 3. 已分配分数样式优化
发酵仓卡片底部的"已分配"显示格式调整为:
```
仓内人员总分 + 跨仓扣分 + 该仓的考核总分数 / 总分
```

示例: `45.5 + 2.0 + 3.0 / 50.0`

## Acceptance Criteria

- [ ] 考核面板的保存按钮默认为灰色禁用状态
- [ ] 当考核分数输入框有变更时,保存按钮变为可点击状态
- [ ] 点击保存后,保存按钮恢复为灰色禁用状态
- [ ] 保存后在人员行下方显示原始/考核/最终分数的摘要信息
- [ ] 已分配分数显示格式符合新要求: `仓内总分 + 跨仓扣分 + 考核总分 / 总分`
- [ ] 所有样式符合 ucss 规范(仅使用类选择器和 flex 布局)
- [ ] 代码通过 lint 和 typecheck

- 使用 UTS 语言,遵循强类型规范
- 使用 `type` 而非 `interface` 定义类型
- 使用 `null` 而非 `undefined`
- 条件语句必须使用布尔值
- 样式仅支持 flex 布局和类选择器
